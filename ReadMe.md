## 🌱 Repository Organization

To keep the repository modular and easy to maintain, each major topic is managed in its own Git branch instead of placing all code in the main branch.

This approach allows every module to have an independent development history while keeping the repository well organized.

### Current Branches

| Branch                           | Description                                                                             |
| -------------------------------- | --------------------------------------------------------------------------------------- |
| `main`                           | Repository documentation and overview                                                   |
| `Core_JAVA`                      | Java fundamentals, OOP concepts, Collections, Exception Handling, and practice programs |
| `Data_Structures_And_Algorithms` | DSA implementations, coding challenges, and algorithm practice                          |
| `Design_Patterns_And_Principles` | SOLID principles, design patterns, and implementation examples                          |

> Additional branches will be added as new modules are completed during the training.

## 💡 Why a Branch-Based Structure?

Organizing the repository using dedicated branches provides several benefits:

* Better separation of learning modules
* Cleaner repository management
* Independent version control for each topic
* Easier navigation between technologies
* Scalable structure for future additions
